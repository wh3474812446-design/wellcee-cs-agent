# 上线与迭代规划（一页纸）

> Step 8 交付物 ｜ 2026-07-08 ｜ 前置：Demo 已上线（评测 94%/红线 0）、PRD v1.0、评测规范与工具链就绪
> 原则：**先在最贵、最容错的流量上验证，用数据换放量**。预期锚点：Airbnb AI 客服灰度多年后线上解决约 40% 咨询——我们评测集口径 94% ≠ 线上口径，灰度期以 40-55% 为现实预期起点。

## 1. 灰度三阶段

| 阶段 | 范围 | 为什么 | 准入条件 | 放量/回滚条件 |
|------|------|--------|---------|--------------|
| ① 英文用户 FAQ 类（2 周） | 英文用户的 faq_direct/chitchat 分支；guide/tool/human 全部直通人工 | 英文人工客服最贵→省得最多；英文用户量小→风险可控（评审 R5 的反直觉结论） | 全量 50 条二轮评测 ≥96%；法务过一遍红线话术；工单邮件通道就绪 | 放量：自动解决率 ≥40% 且幻觉投诉 0 → 进②；回滚：任一幻觉投诉成立 → 该意图降级直通人工 |
| ② 全量 FAQ + 流程引导（4 周） | 中英全部用户；开放 guide 分支与澄清反问 | FAQ/引导覆盖了意图表中 15/20 的 MVP 意图 | ①阶段 CSAT ≥75%；知识库缺口 G01-G07 至少补齐 P0 的 4 项（真实口径替换 🔶 假设值） | 放量：解决率 ≥50%、转人工率 ≤45% → 进③；观察：unclear 触发率>15% 则实施拍板 A 完整版（2 次澄清计数） |
| ③ 工具调用接真实系统（持续） | 押金状态查询接真实 API；工单写入接口化 | mock 已验证交互设计，接口是纯工程项 | 后端提供押金查询/工单写入 API；接口联调通过评测集 tool 类回归 | 最终态指标：北极星自动解决率 ≥55%（PRD 目标）、工单字段齐全率 ≥90% |

## 2. 监控指标与报警（上线即埋，P0 埋点四件套）

| 指标 | 口径 | 目标 | 报警阈值 |
|------|------|------|---------|
| 自动解决率（北极星） | 解决+正确转人工 / 总会话 | 阶段①≥40% → 最终 ≥55% | 日环比降 >10pp |
| 转人工率 | 转人工会话占比（含主动点按钮） | ≤35%（最终） | 单意图转人工率突增 >20pp（疑分类劣化） |
| 幻觉投诉 | 用户投诉+抽检中 D 维不通过 | **0 容忍** | ≥1 例即当日复盘，涉事意图临时降级 |
| CSAT | 会话结束 👍 占比 | ≥80% | 连续 3 日 <70% |
| 首 token 时长 | 用户发问→首字 | <3s | p95 >5s（查供应商/检索延迟） |

## 3. 数据飞轮（周迭代，链路已实测验证）

```
线上对话抽样（每周 100 条 + 全部 👎 与转人工会话）
  → 按《评测规范》四维标注（客服组长每周 1 小时批优先级）
  → 归因四分法分派修复：
     知识库缺口→补条目 ｜ 检索未命中→调参/补问法变体 ｜ 意图误判→改分类例句 ｜ prompt 缺陷→改节点规则
  → 编排即代码重推（本项目实测：改 prompt→重新发布 10 分钟）
  → badcase 进回归评测集（评测集随线上问题持续长大）
  → 周版本发布 + 指标对比留痕
```

关键机制：**AI 答不上的问题自动进缺口清单**（本项目建库阶段已登记 11 条），人工只做优先级判断——评测工具链（run_eval.py + 打分 SOP）直接复用为周回归工具，单轮 50 条全流程约 40 分钟。

## 4. ROI 测算 🔶（上线前以真实账单与工单量校准）

| 项 | 数值（假设标注） | 依据 |
|-----|----------------|------|
| LLM 成本/会话 | ≈ ¥0.02-0.05 | DeepSeek v4-flash/pro 混用，平均 3 轮×约 7K tokens🔶 |
| 人工成本/会话 | ≈ ¥3-7 | 客服时薪 25-40 元🔶 × 单会话 5-10 分钟；英文客服更高 |
| 单会话净省 | ≈ ¥3-6.9 | 人工成本 − AI 成本（AI 解决的会话） |
| 日咨询量假设 | 1,000 会话🔶 | 千万级注册用户，待工单数据校准 |
| **月度净省（50% 拦截）** | **≈ ¥4.5万-10万** | 1000×50%×省¥3-6.9×30 天 |
| 额外收益（未计价） | 转人工工单字段齐全率 ≥90% → 人工单均处理时长下降；7×24 即时应答 → 租房高时效场景的转化挽回 | 评审 C3 与 PRD 问题陈述 |

**一句话结论**：即使按最保守口径（40% 拦截、单会话省 ¥3），月净省也在 ¥3.6 万量级，远超 LLM 成本与维护人力；且 ROI 的大头其实在"英文客服人力释放"与"工单质量提升"这两项未计价收益上。

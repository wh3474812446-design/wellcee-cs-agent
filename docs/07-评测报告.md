# Wellcee 智能客服 Agent 评测报告

> Step 7.6 交付物 ｜ 2026-07-08 ｜ 评测对象：小Cee Demo（Dify Chatflow，DeepSeek v4）
> 配套：《评测规范.md》《评测集.xlsx》（3 sheets）《badcase分析.md》｜ Demo：https://udify.app/chat/ouNOkmWogUiuEgKf

## 1. 摘要

对已上线的小Cee 智能客服 Demo 进行了两轮评测：**首轮 50 条自动解决率 94%（47/50），幻觉与越权承诺 0 例，7 条必转人工场景全部正确路由**。3 个未解决 badcase 经归因全部收敛到「意图分类例句缺口」与「prompt 规则位置」两类可运营根因，修复仅改动分类描述与 2 处 prompt 规则（未动模型与架构），二轮复验 8/8 通过且无回归。系统达到灰度上线的质量门槛；遗留项见第 6 节。

## 2. 评测方法

- **评测单位**：单轮「用户输入 → Agent 完整回复」，记录完整路由路径（流经节点序列）与回复全文
- **判定标准**：《评测规范 v1.0》四维二值判定——A 正确性 / B 完整性 / C 语气合规 / D 安全性（红线维度，单票否决）；整体判定「解决 / 正确转人工 / 未解决」，自动解决率 =（解决+正确转人工）/ 总数
- **评测集**：50 条五类，按真实咨询分布与风险场景设计：

| 类别 | 条数 | 设计意图 |
|------|------|---------|
| 标准问 | 15 | 覆盖 MVP 高频意图的规范问法 |
| 口语化变体 | 10 | 错别字（压金）、方言腔（咋整）、中英夹杂、情绪化标点 |
| 英文提问 | 10 | 双语能力与来源标签语言一致性 |
| 知识库外 | 8 | 测幻觉：发票/补贴/违约金等库内无答案的问题 |
| 必转人工 | 7 | 测边界：举报/纠纷/资金损失/投诉/显式要人工/注销/辱骂诱饵 |

- **跑测方式**：浏览器自动化——脚本经 CDP 直连已登录会话，逐条调用 Dify draft run API（流式），自动记录节点路径与回答（`_tools/run_eval.py`，单条模式规避 30s eval 超时）
- **打分方式**：依规范逐条人工判定（LLM 评委按 SOP 执行），不通过必附理由与原文引用，结果全量留痕于《评测集.xlsx》

| 类别 | 解决 | 未解决 | 说明 |

### 3.3 设计假设的数据验证（面试讲述点）

1. **高风险关键词规则前置于 LLM 分类**（评审修订 R1）：T01/03/04/05 未经分类器直达收集流程——确定性路由在最不能出错的场景兜住了底
2. **「仅基于检索结果回答」+ 诚实兜底话术**：8 条库外陷阱全部守住，包括最易编造的英文违约金问题（H05）
3. **转人工=产品功能**：T02 纠纷场景首轮即开始结构化收集（订单号），符合「工单必要字段齐全率」设计意图
4. **预埋诱饵 T07**（辱骂 AI，且不含任何高风险关键词）：chitchat 分支道歉+主动给人工路径，不还嘴不辩解——验证了兜底链路在词表之外仍安全

## 4. Badcase 归因（四分法）

3 个未解决 + 4 个轻微缺陷，全部归因（明细见《badcase分析.md》）：

| 归因 | 数量 | 代表 case | 修复 |
|------|------|----------|------|
| ③ 意图误判 | 3+2 轻 | V02/E10「房东不回复」口语与英文版被 human_handoff 吸附；V07「没人看」被理解为状态故障 | F1：分类器例句增补 + human 类排除说明 |
| ④ prompt 缺陷 | 2 轻 | T03 资金损失红线话术漏出；H04 来源标注失实 | F2/F3：来源规则收紧、资金损失规则前置加粗 |
| ② 检索变体缺口 | 1 辅因 | V07 的 KB020 问法距离远 | F4：知识库 v1.1 补问法变体（待办） |
| ① 知识库缺口 | 0 | — | 缺口清单机制在建库阶段已前置消化 |

**根因洞察**：所有 badcase 都收敛在「分类例句/prompt 规则」这一可运营层面，无一需要动模型或架构——意图分类体系与红线设计把问题的爆炸半径控制住了，这正是数据飞轮能以周为单位迭代的前提。

## 6. 遗留问题与下一步

1. **全量二轮**：本轮复验为定向 8 条；正式上线前建议全量 50 条重跑一轮取得完整二轮指标（预估 ≥98%）
2. **KB020 问法变体**（F4）：知识库 v1.1 待办——「没人看/没曝光/浏览少」等变体入库，降低对分类器的依赖
3. **拍板 A 完整实现**：澄清反问「最多 2 次、第 2 次按钮式」的会话计数版（Demo 当前为 1 次按钮式简化）；本轮 unclear 触发 0 次，暂无数据支撑加急
4. **评测集扩充**：多轮对话场景（转人工收集完整链路）、繁中问法，纳入周迭代评测集
5. **口径说明**：94% 为评测集口径（题目按意图分布设计）；线上真实解决率受长尾问法影响会更低，以灰度数据为准校准
